The Kamae 3 document: the polished project scope, written as if the project already exists and publishable on day one of the repository. Its stem—what the project is for, who it serves—comes from the seed; its technical content from the System Design. A forcing function for scope: if you can't describe installing and using the finished thing, you don't yet know what you're building. Living-continuous, edited freely as the work evolves.

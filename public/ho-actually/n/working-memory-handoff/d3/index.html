The detailed contents of the state-memory file that let the next session pick up exactly where the build left off.

The body of the state memory (Kamae 6): the dense material a fresh agent reads to reconstitute a build's whole state without opening code — sealed decisions, a per-ho log, do-not-rediscover traps, queues, and the practitioner's voice verbatim. Grows by event-gated accretion (light under active oversight, carrying the build when autonomous); the state-memory file that holds it is always present. Hot, non-canonical; kept honest by the freshness and hot / cold rules.

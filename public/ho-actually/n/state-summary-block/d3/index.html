state-summary block

definition
depth 2/3 − deeper + surface

A short, fixed four-line status — done, next, blockers, and how far along — written at the end of every work session.

The four-field block emitted at every ho close and every session end: COMPLETED, NEXT, ACTION ITEMS / BLOCKS, PROJECT LIFECYCLE. Fixed labels and fixed order, so it is both human-glanceable and machine-parseable — the universal minimum of cross-session continuity and a hook surface for automation.

# ho-00 — Orientation: Ho System, Actually

## What this project is

The second instance of `*, Actually` — the Ho System's own vocabulary and
methodology, decomposed into a navigable, depth-layered, terminal-aesthetic
knowledge graph. It consumes the extracted [`star-actually` engine](../../engine)
and supplies only the domain: `nodes/` + `site.yaml`.

## Why it exists

`hosystem` (the dharma Eleventy site) carries a *model* of the `*, Actually`
node — the format adopted as a contract, rendered in hosystem's own paper idiom
(design session 13). This project is **the real thing**: the actual engine, the
terminal aesthetic, spatial navigation, the depth dial — the Ho System as a
walkable graph, not a staged rendering.

## Lineage

- Engine: `star-actually` (the domain-blind Loom + Terminal + Receptionist).
- Sibling instance and exemplar: `ssh-actually`.
- Content source: the Ho System framework —
  `sageframe-no-kaji/ho-system` — its `framework/glossary.md` (69 terms) and
  `framework/` documents.

## Where it stands: the seed

A **10-node seed graph** is live and builds strict-clean (validate: 10 nodes,
graph sound; build: 42 pages, 30 fragments; `make verify` green). Nodes:
`what-is-ho-system` (root), `ho`, `kamae`, `shu-ha-ri`, `mind-hand`,
`agent-task`, `verification-validation`, `forward-only`, `tiered-understanding`,
`devlog`.

Every seed layer is **extracted** — the practitioner's own words from the
glossary and `the-ho-system.md`, provenance-marked, at depths 1–3. No
synthesized prose has been written or committed; the content-track plan
(`kamae-4-content-track-overview.md`) is where that authoring — and the
practitioner's voice-audit — happens.

## Not done (by design)

- The full graph (see the content-track overview): ~69 glossary terms + framework
  docs, carried to depths 4–5.
- Any **synthesized** prose (depths 4–5). It ships only after voice-audit.
- The receptionist (`/ask`) — an engine concern, wired later; first deploys are
  static, degrading to full-text search.
- A git remote and deploy — the practitioner's call.
